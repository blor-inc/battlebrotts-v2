## 4. Code-Change Review (verification lens)

Diff summary: `godot/ui/shop_screen.gd` +28 / `godot/tests/test_sprint17_1_shop_scroll.gd` +228 / `godot/tests/test_runner.gd` +1. No production code touched outside `shop_screen.gd`.

Fix approach (read-through, not code-review — Boltz owns that):
1. New `_saved_scroll_v: int` instance var.
2. `_build_ui()` captures `ScrollArea.scroll_vertical` BEFORE tree teardown.
3. `call_deferred("_restore_scroll")` at end of `_build_ui()` restores after layout settles.
4. `_restore_scroll()` writes back `_saved_scroll_v`; Godot's `ScrollContainer` auto-clamps to `[0, max_scroll_v]` so stale/absurd values are safe (AC-5 confirms).
5. `scroll_deadzone = 0` also set (design §3.2 — wheel step left at engine default, intentional rejection of custom multiplier).

The defer-one-frame pattern is the correct engine-idiomatic fix: `ScrollContainer.max_scroll_v` isn't finalized until the new `VBoxContainer` computes its minimum size, so restoring inline would clamp against stale bounds.
